# plucked-string-waveguide
This is a [digital waveguide model](https://llllllll.co/uploads/short-url/beKGony8hlPZBQtLxhm8E5KeCNe.pdf) of a plucked, vibrating string. Developed by Tarun N. You can copy and paste the raw code from the .dsp file into the online [FaustIDE](https://faustide.grame.fr/) to play the polyphonic instrument with a MIDI keyboard and a browser that supports webMIDI (such as Chrome). Born from an intense love for musical acoustics, this project is the first in my exploration of this area of applied mathematics and signal processing. No AI/LLM assistance was used in developing this program.

In essence, the output signal of the digital waveguide algorithm is the input signal (i.e. a noise burst) summed (i.e. additively "mixed") with a delayed version of the output (i.e. the output from a previous time sample). So given an input signal x(n) at time sample n, the output is y(n) = x(n) + y(n - L), where y(n - L) is the prior output from L samples ago. The value of L determines delay duration and therefore perceived pitch. It is a simplified model of string displacement waves that propagate from the plucked point of the string, and then travel down the length of the string and back to the pluck/listening point. Delay length corresponds to twice the string length.  

There are two papers I referenced heavily to write this digital waveguide. The core string model is from [Combining digital waveguide and functional transformation methods for physical modeling of musical instruments](https://home.mit.bme.hu/~bank/publist/aes22.pdf), by Lutz Trautmann et al. The mixing and balancing of the two string polarizations models is derived from [Plucked-String Models: From the Karplus-Strong Algorithm to Digital Waveguides and Beyond](https://www.jstor.org/stable/3681155?origin=crossref), by Matti Karjalainen et al.
